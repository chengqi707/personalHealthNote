# 个人就医记录APP PRD（适配Claude Code开发，本地存储+增删改查+健康评估）
## 一、产品定位
一款服务于个人的**本地就医记录管理APP**，核心数据存储在本地（无网络也可正常使用），核心用于记录个人就医全流程关键信息，支持对就医记录的增、删、改、查，同时可将就医记录转换为固定格式上传至云侧模型，获取健康状态评估及生活建议，操作简单、界面简洁，贴合国内用户使用习惯，方便用户快速录入、查询过往就医信息，用于后续复诊参考、医保报销辅助、健康管理等。此外，APP还包含用药提醒（独立Tab）和用户登录+云端数据同步（扩展功能，低优先级迭代）等模块。

## 二、核心功能清单（明确可开发，无冗余）
1. 就医记录增删改查：新增就医记录、删除单条/多条记录、编辑已有记录、查询所有记录（支持按时间筛选）；
2. 记录核心信息录入：包含就医时间、就诊地点（医院）、接诊医生、症状描述、就诊结果、检查项目、药品清单；
3. 本地数据存储：所有记录持久化存储在本地（优先SQLite，适配移动端），无网络也可正常使用；
4. 基础辅助功能：记录排序（按就医时间倒序）、必填项校验、删除/编辑确认提示、空数据提示；
5. 就诊记录格式标准化：端侧自动将单条就医记录的所有字段转换为固定格式的文本，作为云侧模型的输入内容；
6. 云侧模型API调用：实现异步调用云侧模型接口，上传标准化文本，获取返回的健康状态评估+生活建议；
7. 评估结果展示：在就医记录详情页新增功能入口，调用成功后弹窗展示完整的评估与建议内容；
8. 结果本地缓存：将模型返回结果与对应记录绑定，本地缓存（无过期时间），避免重复调用API；
9. 基础交互反馈：展示API调用加载状态、调用成功/失败提示，无复杂交互逻辑。

## 五、交互流程（完整链路，贴合国内APP操作习惯，供CC开发参考）
### 5.1 核心交互链路（增删改查）
1. 新增记录：首页 → 点击底部“+” → 新增页填写表单 → 点击提交 → 校验通过 → 保存到本地 → 跳转首页 → 提示“新增成功”；
2. 查看记录：首页 → 点击单条记录 → 进入详情页 → 查看完整信息；
3. 编辑记录：详情页 → 点击右上角“编辑”/底部“编辑” → 进入编辑页 → 修改内容 → 点击“保存” → 校验通过 → 更新本地数据（清空评估缓存） → 跳转详情页 → 提示“修改成功”；
4. 删除记录：
    - 方式1：首页 → 长按单条记录 → 弹出“删除” → 点击删除 → 确认弹窗 → 确定 → 删除本地数据 → 首页刷新 → 提示“删除成功”；
    - 方式2：详情页 → 点击底部“删除” → 确认弹窗 → 确定 → 删除本地数据 → 跳转首页 → 提示“删除成功”；
5. 查询/筛选：首页 → 点击右上角“筛选” → 选择筛选条件（全部/近1个月/近3个月/近1年） → 列表实时刷新，显示符合条件的记录（默认按就医时间倒序排列）。

### 5.2 健康评估核心链路
详情页 → 点击“生成健康评估” → 校验：
- 无网络：提示“网络未连接，无法生成健康评估” → 终止流程；
- 记录无有效信息：提示“记录无有效信息，无法生成评估” → 终止流程；
- 校验通过：显示加载状态 → 检查本地缓存：
  - 有缓存：直接展示缓存结果弹窗；
  - 无缓存：端侧将记录转换为固定格式文本 → 异步调用云侧模型API → 解析结果：
    - 调用成功：更新本地缓存 → 展示结果弹窗；
    - 调用失败：提示“评估生成失败：XXX”（如超时、接口异常）→ 恢复按钮状态。

### 5.3 辅助交互规则
1. 输入框交互：点击输入框弹出键盘，失去焦点时收起键盘，支持换行（多行输入框）、光标定位；
2. 时间选择交互：点击时间控件 → 弹出系统日期时间选择器 → 选择完成后自动填充到输入框；
3. 提示交互：所有操作结果（新增、修改、删除成功/失败）均用Toast提示（底部显示，时长1.5s）；
4. 弹窗交互：确认弹窗（删除、放弃修改）均为居中弹窗，黑色半透明背景，按钮横向排列（取消、确定），点击取消关闭弹窗，点击确定执行对应操作；
5. 空状态交互：首页无记录时，点击“+”按钮可直接跳转至新增页；
6. 键盘适配：所有表单页面支持滚动，避免键盘遮挡输入框（新增/编辑页）；
7. 健康评估按钮交互：记录信息为空时禁用按钮，无网络时点击提示网络异常，加载状态下禁用点击。

## 六、数据结构（本地存储，供CC生成实体类和数据库）
### 6.1 核心实体类（就医记录）
```kotlin
// 就医记录实体类（Kotlin/Java 均可，CC可自动转换）
data class MedicalRecord(
    val id: Long,                     // 唯一标识（自增，用于删除、编辑）
    val medicalTime: String,          // 就医时间（格式：yyyy-MM-dd HH:mm）
    val hospital: String,             // 就诊医院（必填）
    val doctor: String,               // 接诊医生（非必填）
    val symptoms: String,             // 症状描述（必填）
    val diagnosisResult: String,      // 就诊结果（必填）
    val checkItems: String,           // 检查项目（非必填）
    val medicines: String,            // 药品清单（非必填）
    val createTime: Long,             // 创建时间（时间戳，用于排序）
    // 新增健康评估相关字段
    val healthEvaluation: String? = null,  // 健康状态评估结果（本地缓存）
    val lifeSuggestion: String? = null     // 生活建议结果（本地缓存）
)
```

### 6.2 本地存储说明
1. 存储方式：优先使用SQLite数据库（移动端本地存储最优），创建数据表`medical_record`，字段对应上述实体类；
2. 排序规则：默认按`createTime`倒序排列（最新记录显示在最前面）；
3. 筛选逻辑：根据`medicalTime`筛选（对比时间戳，筛选对应时间段内的记录）；
4. 数据持久化：APP卸载后数据丢失，无需云端同步，无备份功能（简化开发）；
5. 缓存逻辑：仅在首次调用API成功后，将`healthEvaluation`和`lifeSuggestion`字段与记录绑定存储，无过期逻辑；记录编辑后清空该字段，删除记录时同步删除缓存数据。

```
1. 就医时间：${medicalTime ?: '未填写'}
```

### 6.4 云侧模型API调用规范
#### 6.4.1 基础信息（可根据实际模型调整）
1. API端点：`${YOUR_MODEL_API_URL}`（需替换为实际云侧模型接口地址）；
2. 请求方式：POST，Content-Type：`application/json;charset=utf-8`；
3. 超时时间：30秒。

#### 6.4.2 请求参数构造

```json
    "content": "${转换后的固定格式文本}"  // 核心参数：端侧转换后的就诊记录文本

```

#### 6.4.3 返回结果解析（假设模型返回JSON格式，可根据实际调整）
模型返回示例：

- 无需加密传输，直接明文调用接口。

## 七、异常处理（简单落地，避免开发复杂逻辑）
### 7.1 原有功能异常处理
1. 必填项未填写：提交/保存时，提示对应错误（如“请填写就诊时间”），对应输入框边框变红；
2. 输入内容过长：超过最大输入字数（症状/就诊结果500字，检查/药品300字），禁止输入，弹出提示“输入内容已达上限”；
3. 时间选择异常：禁止选择未来时间（就医时间不能晚于当前时间），选择未来时间时提示“就医时间不能为未来时间”；
4. 删除/编辑异常：无对应记录时，弹出提示“记录不存在”，跳转回首页；
5. 数据库异常：本地存储失败时，弹出提示“保存失败，请重试”，可重新提交；
6. 空操作异常：无记录时，点击删除/编辑按钮，无响应，不弹出弹窗。

### 7.2 健康评估功能异常处理
1. 无网络：点击“生成健康评估”按钮后Toast提示“网络未连接，无法生成健康评估”，恢复按钮状态；
2. API调用超时：提示“评估请求超时，请稍后重试”；
3. API返回异常（非200状态码）：提示“评估生成失败：接口返回异常”；
4. 结果解析失败：提示“评估结果解析失败，请重试”；
5. 记录编辑后缓存清空：编辑记录并保存后，自动清空该记录的评估缓存字段，下次点击重新调用API；
6. 空返回结果：提示“未生成有效评估结果，请重试”；
7. 记录无有效信息：点击按钮后Toast提示“记录无有效信息，无法生成评估”，按钮保持禁用状态。
